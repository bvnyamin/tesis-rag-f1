# Collaboration Guide

## Branching

- Usar ramas cortas por feature: `feature/...`, `fix/...`, `docs/...`.
- Mantener `main` estable.

## Commits

- Hacer commits pequenos y descriptivos.
- Preferir mensajes en formato imperativo.

## Pull requests

- Explicar objetivo, cambios principales y forma de probar.
- Evitar mezclar infraestructura, datos y logica en la misma PR cuando no sea necesario.

## Convenciones iniciales

- Python como fuente principal en `src/`.
- Notebooks solo para exploracion.
- Datos pesados fuera del repositorio salvo muestras pequenas o metadatos.
